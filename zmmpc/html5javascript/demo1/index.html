setinterval－requestAnimation

应用规则：我们把球移动到新的位置，新的图像得以创建，但是没有机会将他显示出来，因为帧结束的时候并没有将对象绘制canvas上，为实现动画，我们需要执行以下操作

  • 1:执行帧所要调用的代码
  • 2:将所有对象绘制到canvas上
  • 3:重复这一过程渲染下一帧

setinverval他无法达到毫秒级的精确度（每个浏览器的定时分辨率不同）无法使用它做高质量的动画，同一时刻在浏览器的任务队列中有其他的任务话，动画代码的执行任务将不得不等待在哪里

window.requestAnimationFrame函数接受一个回调函数作为参数，并确保重绘屏幕前执行该回调函数，在某些浏览器中接受第二参数，可以通过他制定的一个html元素提供的动画的可视区域，在回调函数中对程序的修改定发生在下一个浏览器重绘之前，可以通过对requestAnimationFrame函数的链式调用实现动画循环. 

				  	  (function draw(){
				  	  	  window.requestAnimationFrame(draw,canvas);
				  	  	  //animation code..
				  	  })();

由于requestAnimationFrame各个浏览器对它的支持还有待兼容，如果希望跨平台，可以使用如下的代码可以用来规范改函数在不用浏览器中的表现 

  
				  	    if (!window.requestAnimationFrame) {
  								 window.requestAnimationFrame = (window.webkitRequestAnimationFrame ||
                                  window.mozRequestAnimationFrame ||
                                  window.msRequestAnimationFrame ||
                                  window.oRequestAnimationFrame ||
                                  function (callback) {
                                    return window.setTimeout(callback, 17 /*~ 1000/60*/);
                                  });
